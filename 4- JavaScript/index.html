JavaScript Notları

Giris

  • JavaScript kodlarimizi internal olarak yazmak istersek body etiketi disina, html etiketi icerisine script etiketi arasina yerlestiririz.
  • Eger External olarak yazmak istersek, script.js adinda bir dosya olustururuz. Olusturdugumuz bu dosyayi html sayfasina dahil etmek icin de script:src kisayolunu kullaniriz. Src olarak da olusturdugumuz js dosyasini gosteririz.
  • Js dosyasinda yazdigimiz kodlari Chrome tarayicisi uzerinde F12-Console araciligiyla kolaylikla test edebiliriz. Bunun icin js dosyasinda kullanacagimiz kodlar;
    • console.log() >> Icine verilen bir degeri console ekranina yazdirir.
    • console.error() >> Icine verilen bir degeri console ekranina error seklinde yazdirir.
    • console.warn() >> Icine verilen bir degeri console ekranina uyari seklinde yazdirir.
    • console.clear() >> Console ekranini temizler.

Degiskenler

  • Degisken tanimlamak icin 3 adet anahtar kelime vardir.
    • var
    • let
    • const
  • var ile let arasinda sadece scope olarak fark vardir. Buna ileride deginecegiz.
  • const ise sabit olarak nitelendirecegimiz, sonradan degistirilmesini istemedigimiz degiskenler icin kullanilir.
  • En basta hicbir deger atanmamis fakat tanimlanmis degiskenler undefined olarak tanimlanirlar.
    var age; (undefined)

Degisken Tipleri

  • Degiskenler Primitive ve Reference(Object) olarak iki sekilde nitelendirilirler.
  • Degiskenin tipini ogrenmek icin console.log(typeof degisken_ismi) seklinde bir kullanim ile degiskenin tipini console ekranina bastirabiliriz.
  • Primitive Types
    • String > var name = "enes";
    • Number > var age = 22; or var weight = 66.5;
    • Boolean > var isActive = true;
    • null > var surname = null;
    • undefined > var height;
  • Reference Types - Objects
    • Array > let names=['Ali','Atilla','Alper']
    • Object > 
      let address = {
        country : 'Turkey'
        city    : 'Izmir',
    }
    • Function > 
      var calculateAge = function(){
        return 0;
    };
  • Burda en onemli farklardan bir tanesi integer ve float gibi farkli sayisal degerlerin olmamasidir. Tek bir sayisal tur var o da number.

Degisken Donusumleri

  • Number() seklinde bir fonksiyon kullanarak belirli degerleri numbera cevirebiliriz.
  • String() seklinde bir fonksiyon kullanarak belirli degerleri stringe cevirebiliriz.
  • (10).toString(); seklinde tur icinde tanimlanmis fonksiyonlari kullanarak da string donusumu yapialbilir.
  • val = parseInt("10"); seklinde kullanim ile Stringleri Numbera cevirebiliriz.
  • val = parseFloat("10.5"); seklinde kullanim ile Stringleri Float seklinde kullanilabilir bir numbera cevirebiliriz.

Operatorler

  • Burada 4 farklı operatör türü vardır.
    • Aritmetik Operatorler
    • Atama Operatorleri
    • Karsilastirma Operatorleri
    • Mantiksal Operatorler
  • 1 - Aritmetik Operatorler
    • +,-,*,/,%,++,--
    • Kullanimlari diger dillerle aynidir.
  • 2 - Atama Operatorleri
    • =, +=, -=, *=, /=, %=
    • Kullanimlari diger dillerle aynidir.
  • 3 - Karsilastirma Operatorleri
    • == (Sadece barindirdigi degeri kontrol eder),=== (Hem deger hem de type kontrol eder), !=, !==, >turevleri
    • Kullanimlari diger dillerle aynidir.
  • 4 - Mantiksal Operatorler
    • && And
    • || Or
    • ! Not
    • Kullanimlari diger dillerle aynidir.

Veri Yapilari

  • Bu konu altinda Date, Number ve Stringlerin metodlari ogretildi. Farkli olarak Template Literals verildi.
  • Template Literals
    • Parcali bir sekilde string olustururken kolaylik saglar. String formatlamaya benzer.
    • Normal: 
                              var city = 'Denizli';
                        let val = city + 'is the most beautiful city in the Turkey.';
    • Template Literals 
                                  var city = 'Denizli';
                            let val = `${city} is the most beautiful city in the Turkey.`;
    • Burada onemli noktalardan bir tanesi ' isareti degil, ` isaretinin kullaniliyor olmasi.

Kosul Ifadeleri

  • Switch-case yapisi da Java'daki yapi ile birebir ayni.

Loops

  • for, while, do-while donguleri Java ile birebir aynidir.
  • for dongusunde i tanimlamak icin let kullanmak gerekir. 
                        for(let i=0; i>10; i++){
                        console.log(i);
                    }
  • Dongulerin Object ve Array ile Kullanimi
    • Normal for Kullanimi 
                                  for(let i=0; i>10;i++){
                                console.log(people[i]);
                            }
    • for in Kullanimi 
                                  for(let i in people){
                                console.log(i);
                            }
    • foreach Kullanimi
      Burada function parametresi icerisindeki degiskene people dizisi icerisindeki her bir eleman teker teker kopyalanir. Kopyalanan her bir eleman function icerisinde yazilan isleme tabii tutulur. 
                                  people.forEach(function(item){
                                console.log(item);
                            });
  • Eger object icerisindeki bilgileri geri array olarak almak istersek map fonksiyonunu kullanmamiz gerekir. 
                        let val = people.map(function(item){
                        return item.firstName + item.lastName;
                    });
    Burada people icerisindeki her elemanin 'firstName' ve 'lastName' ozellikleri alindiktan sonra birlestirerek bir eleman olarak val icerisinde depolanir. people icerisindeki eleman sayisi kadar val icerisinde eleman birikir ve array olarak bize doner.

Functions

  • Iki farkli tanimlama sekli vardir.
    1. Function Declaration 
                        function function_name(parameters){
                        processes...
                    }
    2. Function Expressions 
                        const function_name = function(parameters){
                        processes...
                    }
  • Eksik arguman yollama durumunda undefined tanimlanmamasi icin parametrelerin tanimlandigi yerde default degerler tanimlayabiliriz.
  • Eger alinacak parametre sayisi belli degil ise parametre olarak hicbir sey tanimlanmaz. Function icerisinde arguments ozelligi ile alinan tum parametreler liste halinde bulunur ve bununla rahatlikla islem yapilabilir. 
                        let sum = 0;
                    const sumAll = function(){
                        for(let i=0; arguments.length>i ; i++){
                            sum += arguments[i];
                        }
                    }
                    sumAll(10,20,30,50...);

Window Object

  • Normalde NodeJs icerisinde bulunmayan fakat Browser icerisinde var olan bir objedir.
  • Js icerisinde 'var' ile olusturdugumuz degiskenler, functions ve objects barindirir. Scope kavramindan dolayi let ve const window icerisinde olusturulmaz.
  • Window'un bircok yerde kullanimi vardir.
    • alert > uyari kutucugu cikartir.
    • confirm > evet-hayir kutucugu cikartir. return olarak boolean deger dondurur.
    • prompt > kullanicidan input almak icin kutucuk cikartir.
    • .location ozelligi ile adres cubugundaki adresin detayli bilgilerini alabiliriz.
    • .navigator ozelligi ile tarayicinin detayli ozelliklerine ulasabiliyoruz.
    • .document ile html sayfasinin detayli ozelliklerine ulasabiliyoruz.

Scopes

  • Her dilde oldugu gibi Js'de de global ve local scope bulunmaktadir.

  • Global Scope

    • herhangi bir fonksiyon,dongu ya da if statement blogu icerisinde yer almayan yani disarda tanimlanan degiskenler global degiskenlerdir.
    • var ile tanimlanmis degisken window icerisinde de tanimlanir.
    • let ve const ile tanimlanan degiskenler window icerisinde tanimlanmazlar.

  • Local Scope

    • Js icerisinde local scope islemleri birazcik karisik.
    • Fonksiyon blogu icerisinde tanimlanmis her degisken sadece ve sadece o fonksiyon blogu icerisinde gecerlidir.
    • Normal bir yazilim dilinde block icerisinde tanimlanan block icerisinde kalir mantigi olur. Fakat js'de var ile degisken tanimlamak bu durumu biraz degistirir.
      Eger fonksiyon blogu disindaki herhangi bir block(dongu,if,switch) icerisinde var ile bir degisken tanimlanirsa, tanimlanan o deger block disarisinda da gecerli olur.
    • var ile alakali olan bu durum let ve const icin gecerli degildir. Bu yuzden yazilimcilar genelde let ve const kullanirlar.
    • let ve const, blocklar icerisinde tanimli iseler sadece blocklar arasinda gecerli olurlar. Eger global tanimli iseler windows objesi icinde tanimlanmazlar fakat global olarak islem gorurler.

Document Object Model (DOM)


  • DOM yapisi aslinda parent-child iliskisine benzer yapidaki dallanan bir yapidir. Sayfa etiketlerine erismek istedigimizde Window objesi altindaki document aracigiyla biz istedigimiz etikete istedigimiz islemleri yapabiliyoruz.
  • document altından birçok özelliğe erişim sağlanabilir.
    • document.all -> Sayfada bulunan tüm elementleri getirir
    • document.head -> Sayfanın head kısmını getirir
    • document.body -> Sayfanın body kısmını getirir
    • document.anchors -> Sayfadaki tüm linkleri getirir
    • document.URL -> Sayfanın URL'sini getirir
    • document.domain -> Sayfanın domain adresini getirir
    • Kısacası sayfada bulunan tüm elementler document altındadır ve onun üzerinden ulaşılabilir.
  • document altından seçilmiş elementin attribute değerlerine de ulaşılabilir.
    • document.anchors[0].getAttribute('href');
  • Single Element
    Eger sayfa uzerinden tek bir element secmek istersek kullanacagimiz iki farkli metod var. 
                        document.getElementById()
                    document.querySelector()
    Bu metotlarin kullanimiyla yalnizca bir adet element secebilir ve uzerinde istedigimiz degisiklikleri gerceklestirebiliriz.
    • getElementById fonksiyonu içerisine erişmek istediğimiz elementin Id değerini String olarak (tırnak içerisinde) veriririz. 
                 document.getElementById('header')
    • Ulaştığımız elementlerin alt özelliklerini istediğimiz gibi değiştirebiliriz. 
                                  let val = document.getElementById('header');
                            val.style.fontSize = '45px';
                            val.style.color = 'red';
                            val.className = 'btn-active';   //Classları tamamen değiştirir, yeni bir tane eklemez.
                            val.classList.add('btn-muted'); //Classlara yeni bir tane class ekler
    • Eğer elementin içeriğini değiştirmek istersek üç farklı yöntem kullanabiliriz. innerText,innerHTML,TextNode
      • val.innerText = 'Selam'
      • val elementinin içerisine Selam yazar. Fakat HTML komutlarını algılamaz
      • val.innerHTML = ' < b>Selam< /b> '
      • val elementinin içerisine Bold olarak Selam yazar. HTML komutlarını algılar.
      • TextNode ise innerText ile aynı işlevi görür. İlerleyen konularda belirtiliyor.
    • querySelector, getElementById' e göre daha esnek bir yapıya sahiptir.
    • Gerekli özelliklere göre seçmeler yapıalbilir.
      • document.querySelector('#header') -> id'si header olan elementi seçer
      • document.querySelector('.btn') -> Classı btn olan elementi seçer
      • document.querySelector('li') -> Yakaladığı ilk li elemanını seçer
      • document.querySelector('li:last-child') -> gibi özelliklerin kullanılmasına da olanak sağlar
  • Multiple Elements
    Eger birden fazla elemani secmek ve onlar uzerinde islem yapmak istiyorsak birden fazla kullanabilecegimiz metod bulunmakta. 
                        document.getElementsByClassName() > Class isimleriyle ulasmamizi saglar
                    document.getElementsByTagName()   > Etiketlerin isimleriyle ulasmamizi saglar (div,li...)
    
                    Bu metotlar return olarak HTMLCollection dondurur ve elemanlar arasinda yalnizca for dongusu ile gezilebilir.
    
                    document.querySelectorAll()       > Birkac ozel kullanimi ile yine ayni islevi gorur.
    
                    Bu metod ise return olarak NodeList dondurur. Bu sayede elemanlar arasinda forEach dongusu ile kolayca gezilebilir.
  • Elementler Üzerinde Gezinme
    • .childNodes metodu, textNode'lar dahil tüm alt elementleri NodeList şeklinde döndürür.
    • Code içerisindeki her bir satır aralığı denebilir.
    • .children dediğimizde textNodelar işin içerisine katılmaz. Sadece elementler geri döndürülür.
    • childNodes[0].nodeType ile nodeun hangi tipte olduğunu öğrenebiliriz. Details
    • firstChild ile seçilen element altındaki ilk node alınır. TextNode olabilir.
    • firstElementChild ile sadece ilk element alınır.
    • childElementCount içeride kaç tane element olduğunu söyler.
    • parentNode, parentElement gibi özellikler de mevcut.
    • children[0].nextSibling ile seçilen elementten sonra hanig node varsa onu getirir. nexElementSibling ise sadece bir sonraki elementi getirir.
  • Elemen Silme & Güncelleme
    • Elementin kendisini yada childNode, children özelliğinin indexlenmesini kullanarak bir alt elemanını .remove() metoduyla silebiliriz.
    • removeChild metodu içerisine elementin childrenlarından bir tanesini vererek yine silinmesini sağlayabiliriz. 
                                          let val = document.querySelector('#task-list');
                                    val.removeChild(val.children[0]);
    • removeAttribute metoduyla istediğimiz nesnenin istediğimiz attr. değerini silebiliriz. 
      val.children[0].removeAttribute('class')
    • val.classList.add('new') ile yeni bir class ekleyebilir. Aynı şekilde remove metodu ile çıkartabiliriz.
  • Eleman Oluşturma
    • const li = document.createElement('li') ile li değişkeninin içerisine li elementi atanır.
    • li.className = 'list-group-item' li değişkeninin içerisindeki li elementine class eklenir.
    • li.setAttribute('title','new item') metodu ile li elementinine title özelliği eklenir.
    • const text = document.createTextNode('inner text');
      li.appendChild(text)       metodları sayesinde li değişkeninin içerisindeki li elemanının içerisine 'inner text' stringini eklemiş oluyoruz.
    • Eğer li elemanını ul listesinin içerisine eklemek istersek document.querySelector('#task-list').appendChild(li); metoduyla ekleyebiliriz.

Events

  • Bir Event oluşturabilmemiz için önce eventi gerçekleştirecek elementi getElementById, querySelector yardımıyla seçmemiz gerekiyor.
    let btn = document.querySelector('#btnDeleteAll');
  • Elementi seçtikten sonra btn.addEventListener metodu ile event eklenir.
  • addEventListener metodu iki parametre alır; ilki string olarak eventin türü (click,mouseover..), ikincisi gerçekleştirilecek eylem, fonksiyon. Bu fonksiyonu dışarıdan da yollayabiliriz. 
                                btn.addEventListener('event_turu',function() {
                                Yapılacak işlemler..
                            });
  • Aynı element için, aynı event türünü iki farklı fonksiyon olarak tanımlayabiliriz. btn nesnesine tıklandığında ikisi de çalışır. 
                                btn.addEventListener('click',btnClick);
                            btn.addEventListener('click',btnClick2);
  • function içerisine parametre olarak göndereceğimiz event nesnesi (e) ile birçok işlem yapılabilir. Öncelikle e.preventDefault() ile istenmeyen refresh durumunu çözebiliriz. 
                                btn.addEventListener('event_turu',function(e) {
                                Yapılacak işlemler..
                                e.preventDefault();
                            });
    Ayrıca e.target ile tıklanan nesneyi alabiliriz ve alınan nesnenin özelliklerine de e.target.id gibi erişebiliriz.
  • Mouse Events
    • .addEventListener('click',...)
      click : Tek tıklama hareketiyle tetiklenir.
    • .addEventListener('dblclick',...)
      dblclick : Çift tıklama hareketiyle tetiklenir.
    • .addEventListener('mousedown',...)
      mousedown : Mouse tıklama işlemi ilk aşağı sonra yukarı olarak gerçekleşir. Sol tıka basıldığı anda down olur.
    • .addEventListener('mouseup',...)
      mouseup : Mouse tıklama işlemi ilk aşağı sonra yukarı olarak gerçekleşir. Mousedown olduktan sonra sol tıktan parmağımızı kaldırdığımız anda ise mouseup olur.
    • .addEventListener('mouseenter',...)
      mouseenter : Mouse event tanımlanan nesnenin içerisine girdiği anda mouseenter olayı gerçekleşir.
    • .addEventListener('mouseleave',...)
      mouseleave : Mouse event tanımlanan nesnenin içerisinden çıktığı anda mouseleave olayı gerçekleşir.
    • .addEventListener('mouseover',...)
      mouseover : Mouse event tanımlanan nesnenin içerisine girdiği anda mouseover olayı gerçekleşir. Fakat nesne içerisindeki diğer elemanların üzerine gelinirse mouseout olup hemen tekrar mouseover olur.
    • .addEventListener('mouseout',...)
      mouseout : Mouse event tanımlanan nesnenin içerisinden çıktığı anda mouseout olayı gerçekleşir. Fakat nesne içerisindeki diğer elemanların üzerine gelinirse de direkt mouseout olur.
    • .addEventListener('mousemove',...)
      mousemove : Event atanan nesnenin içerisinde mouseun hareket ettiği her an bu eylem çalışır.
  • Keyboard Events
    • keydown : tuşa basıldığında tetiklenir
    • keyup : tuştan elimizi kaldırdığımız an gerçekleşir.
    • keypress : tuşa basılıp çekilme olayında gerçekleşir
    • focus : input alanına mouse ile tıkladığımızda aktif olur.
    • blur : focus olayının tam tersi, input alanından çıktığımızda gerçekleşir.
    • cut : input içerisinde kesme işlemi yaptığımızda çalışır.
    • paste : input içerisinde yapıştırma işlemi yaptığımızda çalışır.
    • submit : form içerisinde bir buton aracılığıyla submit yapmaya çalıştığımızda gerçekleşir.
    • select : input alanında metnin herhangi bir alanını mouse ile basılı tutup seçtiğimizde gerçekleşir.
  • Event Bubbling & Capturing
    • Eğer bir elementten başlayıp üstteki parentları da dahil hepsine click metodu verdiğimizi düşünürsek, en içteki elemana tıkladığımızda; en içten en dışa doğru bütün click metotları çalışır. Buna Event Bubbling denir.
    • Bu olayın tam tersi Event Capturing'tir. Bunu gerçekleştirmek için de addEventListener metodunun 3. parametresini true olarak veririz. Normalde 3.parametre default olarak false'tur. Çünkü default olarak Event Bubbling desteklenir. Biz bunu true yaptığımızda Event Capturing desteklemeye başlarız. .addEventListener('click',function()...,true);
    • Bu durumları ortadan kaldırmak istersek addEventListener içerisindeki function içerisine event nesnesi (e) yollayıp, bu nesne üzerinden e.stopPropagation() metodu çağrılır.

Storage

  • İki farklı depolama seçeneği vardır. Local Storage ve Session Storage
  • Local Storage
    Değerler url'ye bağlı olarak kaydedilir. Tarayıcı kapanıp açılsa bile bilgiler kaybolmaz, depolanır.
  • Session Storage
    Değerler geçici olarak depolanır. Tarayıcı açılıp kapandığında bilgiler kaybolur.
  • İki storage için de kullanılan metotlar aynıdır; 
                                const firstName = localStorage.setItem('firstName','Enes');
                            setItem metoduyla veri depolanır. Key/Value ilişkisi vardır. İlk parametre key, ikinicisi value'dur.
                        
                                let val = localStorage.getItem('firstName');
                            getItem metoduyla depolanan değer alınabilir.
                        
                                localStorage.removeItem('firstName');
                            removeItem ile depolanan değer silinir.
                            .clear metodu ile de storage tamamen temizlenir.
  • Storage içerisinde array depolamaya çalışırsak arrayin elemanları virgüllerle tek bir string haline getirilerek depolanır. Bu da istemediğimiz bir durumdur. Bu işlemi şu adımlarla çözeriz; 
                                hobies = ['enes','alper','balta'];
                            localStorage.setItem('hobies',JSON.stringify(hobies));
                                Şeklinde kullanarak problemi çözebiliriz.
    
                            Geri almak istersek;
    
                            val = JSON.parse(localStorage.getItem('hobies'));

jQuery

  • jQuery'i iki farklı yolla projemize dahil edebiliyoruz. İlki başka bir web server'ından CDN yardımıyla projeye dahil etmek, ikincisi ise External olarak dosyayı indirip proje dosyasından kaynak göstermek.
