CSS Yazim Sekilleri

Internal CSS
  • CSS Kodlari sadece o html sayfasi icin gecerli olacaksa internal css kullanmak mantiklidir.
  • Internal CSS icerisindeki verilen selector etiketler, sayfa icerisindeki ayni etiketteki tum degerleri degistirirler. Yani toplu bir duzenleme soz konusudur.
  • Head etiketi altina style etiketi olusturup bunun icerisine css kodlarinin yazilmasina Internal CSS denir.
Inline CSS
  • Internal CSS sayfadaki etiketleri toplu bir sekilde duzenleme yapar.
  • Eger toplu duzenlemelerde istemedigimiz bir ozelligi ya da sadece o degere bir css ozelligi atamak istersek Inline CSS kullaniriz.
  • Inline CSS, duzenlenecek etiketin baslangic etiketinde yazilir. Baslangic etiketinin icerisine style="" attr. eklenir. Eklenen bu attr. icerisine internal css te oldugu gibi suslu parantez icerisindeki degerler yan yana yazilir.
  • Inline CSS icerisinde herhangi bir deger Internal CSS icerisinde de tanimlanmis ise Inline icerisindeki deger gecerli olur. Yani Internale gore daha etkindir.
External CSS
  • CSS Kodlarini farkli html sayfalarinda kullanmak istenirse herhangibirisim.css seklinde dosya olusturulup Internal CSS de yazildigi gibi suslu parantezlerle css kodu yazilir.
  • Bu CSS dosyasini html dosyasina eklemek icin head etiketine altina link etiketi eklenir. Ex: link rel="stylesheet" href="style.css"
  • Internal CSS te yazilan kodlar External CSS ten daha onemlidir. Yani her iki kodda da color yer aliyorsa Internal CSSteki deger gecerli olur.

Hepsi birlikte yazilmissa gecerlilik sirasi : Inline CSS > Internal CSS > External CSS

ID ve Class

  • Oncelikle CSS calismalari genelde Internal ya da External yapilir. Inline yapilmasi tavsiye edilmez.
  • Css kodlarina ulasmanin bircok yolu vardir.
    1. Genel etiket yoluyla ozellikleri belirleme(p,li..)
      • p{ color:red; }
    2. . ile baslayarak bir class tanimlayip ozellikleri icerisinde belirtmek. Daha sonrasinda bu ozellikleri kullanacak etiketin baslangicinda class attr. icerisine eklemek
      • (style icerisi) .ornekclass{ font-size:20px; }
        (body icerisi) p class="ornekclass" seklinde kullanilir
    3. Css kodu yazilacak etikete bir unique id atamak ve daha sonrasinda #id seklinde internal css tanimlamak
      • (style icerisi) #IDDegeri{ font-size:40px; }
        (body icerisi) p id="#IDDegeri" seklinde kullanilir
  • 1. Yontem ismi verilen tum etiketleri ayni ozellikler verir
  • 2. Yontem verilen ozellikleri birden fazla etikete atanabilir ve ayni etikete birden fazla class da atanabilir
  • 3. Yontem ise sadece o id'ye sahip etiketi niteler

Selectors

Id , etiket ismi ve class olarak etiketlere ulaşabileceğimiz 3 temel yöntemden başka kullanabileceğimiz belli amaçlara göre hazırlanmış Selector türleri vardır.
Bu selector türlerini şu şekilde gruplayabiliriz ;

  • Group Selectors
  • Attribute Selectors
  • Pseudo Selectors

  • Group Selectors

    • Css Group Selector' larını bir kapsam içinde aradığımız etiketler için kullanıyoruz. Örneğin farklı seviyelerde olan bir < p> etiketini düşünelim ;

      • < div>
        < p id="first">
        < p class="description">
        < /div>
        < p> < /p>


      ve bu etiketlerden sadece div altındaki p etiketlerine ulaşmak istiyorsak bu durumda group selectors' ları kullanmamız gerekiyor.

      div p {
      /* div altındaki tüm p etiketleri */
      }

      div #first{
      /* div altında id' si first olan eleman */
      }

      div .description{
      /* div altında class' ı description olan eleman */
      }

    • Diğer group selector kullanımları ;
      1. (*) => tüm etiketler
      2. (p) => tüm p etiketleri
      3. (div p) => div içindeki tüm p etiketleri
      4. (div,p) => tüm div ve tüm p etiketleri
      5. (div > p) => üst etiketi div olan tüm p etiketleri
      6. (p ~ div)=> p ile aynı seviyede tüm div etiketleri
      7. (p + div)=> p etiketinden sonra gelen aynı seviyedeki div etiketi
    • Ornekler:
      Group.html - Style.css

  • Attribute Selectors

    • Attribute Selector' larını etiketlerde belirtilen attribute özelliğine göre kullanıyoruz.
    • Ornegin;
      < img src="resim.jpg" alt="doğa resimleri">
      img etiketi için src ve alt özelliği bir attribute özelliğidir ve bu img etiketine ulaşabilmek için bu attribute özelliklerini kullanabiliriz.
    • Ornekler :
      Attribute.html -Astyle.css

  • Pseudo Selectors

    • Pseudo Selectors' lar 2 grup altında ele alınıyor.
      • Pseudo Classes
      • Pseudo Elements
    • Pseudo Class Selector' ları html etiketlerinin durumunu kontrol ettikten sonra uygulayacağımız css kodları için kullanılır.
      • Syntax:

        selector:pseudo-class {

        property:value;}

      • < a> etiketi için kullanabileceğimiz pseudo class selector kullanımları ;

        • /* henüz ziyaret edilmemiş link */

          a:link {

          color: blue; }

        • /* ziyaret edilmiş link */

          a:visited {

          color: black; }

        • /* mouse link uzerindeyken */

          a:hover {

          color: yellow; }

        • /* linke tiklama ani */

          a:active {

          color: red; }

    • Pseudo Element Selector' ları ise bir etiketinin her hangi bir parçasını seçmek istediğimizde kullanırız. Pseudo class selector' larından farklı olarak "::" sembolü kullanılır.
      • Syntax:

        selector::pseudo-element {

        property:value; }

      • Örneğin; < p> etiketinin kapsamındaki yazı içinde ilk satırı seçmek istersek ::first-line özelliğini kullanabiliriz.

        • p::first-line {

          color: red; }

    • Ornekler :
      Pseudo.html -Pstyle.css

CSS Inherit

  • CSS Inherit aslinda oop'de gecen inheritence mantigidir.
  • CSS'te en distaki body ye verilen ozellikler altindaki tum etiketler icin gecerli olur.
  • Fakat her bir etikete farkli css kodlari yazmaya baslarsak artik o ozellikler gecerli olur.
  • Yani parentlarin ozellikleri childlara gecer fakat childlarda da ayni ozellikler varsa child icerisindeki ozellikler gecerli olur.
  • Yani CSS inherit de ozel olan kodlar daha onemli ve gecerli olurlar.
  • Etiket Kodlari < Class Kodlari < ID Kodlari
  • Fakat bazi kodlar inheritence durumunu kullanmaz. Mesela a etiketi. Bu deger icin inherit kullanmak istersek a etiketi icin yazilan css kodu icerisine hangi deger inherit edilmek isterse ona inherit degeri atanir.
    Ornek : color=inherit;
  • Eger yazilan degerin kesinlikle calismasini istersek !important anahtar kelimesini kullanarak gecerli kilabiliriz.
    Ornek : color="orange" !important

Floating Divs

  • Burada onemli olan kisim float eklenen divlerin akis disina cikmasi.
  • Bu yuzden varsa container divinin ozellikleri kaybolur, cunku icerisindeki divler akis disina cikmistir.
  • Bunu cozmenin birkac yolu var;
    * container divine clear classi verip internal css tanimlarken clear classina overflow: auto olarak tanimlamak
    * container div icerisine bir tane daha div eklenir. Bu div'e inline olarak clear: both attr. verilirmesi
    * container divine clearfix classi verip internal css tanimlarken .clearfix:after pseudo selectorini kullaniriz. Burada 3 tane ozellik eklenir:
    - content : ""; (Bos bir content tanimlanir)
    - display : block;
    - clear : both;
    Diyerek sorun cozulur.

Display

  • Displayin alabilecegi dort deger vardir
    -none -inline -block -inline-block
  • Inline etiketler icerigi kadar yer kaplarlar ve eklenen her bir etiket yanyana gelmeye devam eder. Fakat yukseklik ve genislik degerleri verilemez. Icerigi kadar alan kaplar.
  • Block etiketler browserin genisligi kadar alan kaplarlar. Yuksekligi icerik kadardir. Her yeni block elemani alt alta siralanir. Genislik ve yukseklik verilebilir.
  • Inline-Block elemanlar ise temel ozellikleri inline gibidir, yanyana dizilirler fakat block gibi genislik ve yukseklikleri ayarlanabilirler.
  • None ise verilen etiketi ekranda gorunmez kilar.

Positioning

  • Bu ozellik istedigimiz etiketi, istedigimiz noktaya yerlestirmemizi saglar.
  • Position ozelligi olarak verilen 5 tane deger vardir:
    -static
    -relative
    -fixed
    -absolute
    -sticky
  • static
    • Bu position ozelligi icin varsayilan degerdir. Ilk atanan halini korur.
  • relative
    • Position ozeliigi relative verilen bi ogeyi dort koseden hizalama yapilabilir.
    • Eger bir nesnenin top ve left degerlerine 30px atanmis ise bu nesnenin yeni konumu, normal akistaki konumu referans alinarak ayarlanir. Sinir cizgileri akistaki ilk konumudur.
  • absolute
    • Absolute yapilan nesne normal akisin disina cikar. Relativeden rakli olarak artik body'e gore hizalanir.
    • Absolute verilen nesne eger bi div icerisindeyse yine body'e gore hizalanir.
    • Eger bulundugu div'e position olarak relative ozelligi verilirse artik icerisinde bulundugu dive gore hiza alir. Kisacasi absolute olan nesne relative bir ust etiket arar ve bulursa ona gore hizalanir, bulamaz ise bodye gore hizalanir.
  • fixed
    • Bu kullanim sitelerdeki Canli Sohbet tarzindaki kutular gibi fixed verilen nesnenin kullanicinin gordugu ekranda surekli sabit kalmasini saglar. Scroll bar kaydirilsa bile goruntu verilen noktada sabit kalir.
  • sticky
    Bunu soyle ozetleyebiliriz; normalde ekran icerisinde yer alan bir eleman scrollbar ile sayfayi kaydirdigimizda kaybolur. Eger bu eleman sticky olarak yerlestirilirse, ekran kaydirildiktan sonra bile verilen konumda tarayici da kalmayadevam eder. Bu ozellik sitelerdeki ustte sabit kalan menuler icin kullanilabilir.

Box Model

  • Margin
    • Margin degeri nesnenin 4 kenarindan etrafina olan boslugu ayarlar.
    • Bircok kullanimi vardir;
      Tek deger verilebilir bu top right bottom ve lefte otomatik olarak atanir. ex: margin : 2px;
      Tek tek degerler de verilebilir. Bu degerler toptan baslayip saat yonune dogru devam eder. ex: margin : 2px(top) 2px(right) 2px(bottom) 2px(left);
      Cifter olarak da verilebilir. ex: margin : 2px(top&bottom) 5px(left&right)
    • Baslangicta browser body etiketine otomatik bir margin degeri atar. Genelde bu en basta sifirlanir.
    • Inline elemanlara ust ve alttan margin etki etmez! Sadece sag ve soldan marginlenebilirler. Fakat bu da inline-block yontemiyle cozulebilir.
    • margin : auto: denirse, verilen etiket browserdan otomatik olarak hesaplanarak mutlaka sayfanin tam ortasina konumlanacak sekilde durur.
  • Padding
    • Padding, icerik ile koseler arasindaki mesafeyi ayarlar.
    • Padding tanimlamalari margin ile aynidir.(Siralamalar vs)
    • Eger padding uygulanacak olan elemanin yukseklik ve genislik degerleri belli ise uygulanan padding degeri kadar yukseklik ve genisligi buyur. Yani padding degerimiz 50px ise ve yukseklik,genislik 100px ise son durumda yuks.&genislik 150px durumunda olur.
    • Eger inline elemanlarda fazla bir padding yaparsak, elemanlar sabit kalir fakat arkaplani diger ogelerinonune gecer. Yani margin gibi eleman asagiya kaymaz.Normal akistaki yerinde sabitlenir.
  • Border
    • Border verilen elemanin cevresinde bir cerceve olusturur.
    • Bircok olusturma yolu bulunur. Direkt olusturulurken 3 degere ihtiyac duyar. boroder : boyutu(px) tipi(solid vs) rengi
    • Her kenara butun ozellikleri tek tek atayabiliriz.
    • border-radius degeri koseleri yumusatmaya yarar. Eger tam yuvarlak yapmak istersek border-radius : 50%; yaparsak nesne yuvarlak olur.
  • Box Model & Box Sizing
    • Box Model elementlerin sayfadaki buyukluk degerleriyle alakali olan temel modeldir.
    • Box Sizing ise bu box modelin nasil hesaplanacagini ayarlar. Iki secenek vardir;
    • Normalde Box Sizing, content-box olarak ayarlidir. content-box ise width ve height degerlerini hesaplarken isin icerisine margin,padding ve border degerlerini de katar. Ornek olarak 300px bir element varsa margin,padding ve border ozellikleri de 10px ise bu elemanin sayfada kaplayacagi alan 360px olur. Bunu padding de son satirda bahsetmistik.
    • Eger box-sizing : border-box; olarak tanimlanirsa, elementin sayfadaki width ve height degerleri elementin buyuklugune ve margin degerine gore hesaplanir. Yani elementin buyuklugu icerisinde hem element hem border hem de padding degeri de bulunmaktadir.Ornek olarak 300px bir element varsa margin,padding ve border ozellikleri de 10px ise bu elemanin sayfada kaplayacagi alan 310px olur. Cunku 300 px icerisinde border ve padding degerleri de bulunmaktadir. Fakat icerisindeki elemente ayrilan alan azalacaktir. 20+20 = 40px azalir 260px kalir.

    • text-align ozelligi standart olarak left, right,center ve justify olarak 4 cesit kullanimi vardir.
    • Buradaki justify ozelligi, metin icerisindeki tum kelimelerin saga ve sola yapismasini saglar. Aralarda gereksiz bosluklar birakilmaz.
    • text-align islemi elementin kapsamina gore hizalar. Mesele a etiketine sahip bir icerigin kapsami icerikteki metin kadardir. Align islemi etki etmez ama a etiektini div icerisine alip div etiketine align right ozelligi verirsek a etiketi icerisindeki icerik en saga kaymis olur. Cunku div etiketi block ozellik gosterir.
    • Eger paragarafi sutunlara bolmek istersek o elemente column-count ozelligini kac sutuna boleceksek o kadar deger atariz. ex: column-count : 2 (2 sutuna boler)
    • Sutunlara boldugumuz elementin sutunlar arasina cizgi eklemek istersek column-rule ozelligine ayni border eklermis gibi ozellikler gireriz. ex: column-rule : 1px solid gray;
    • column-count ozelligi CSS 3 ile gelen bir ozellik. Her tarayicida calistigina emin olmak icin 3 kod da eklenir;
      column-count : 2;
      -webkit-column-count : 2; (opera,chrome ve safari icin)
      -moz-column-count : 2; (mozilla icin)
      Ayni sekilde column-rule icin de hepsi tek tek yazilir.

Responsive Design

  • Responsive Tasarim, mobil cihazlar gibi farkli cozunurluklere sahip olan ekranlarda da sahip oldugumuz temanin duzgun gozukmesini saglamaktir.
  • Bazi siteler mobile subdomaini ile mobile ozgu ikinci bir tema gerektirirken responsive tasarimda boyle bir seye gerek yoktur.
  • Responsive tasarim ana noktalarindan birisi siteye baglanan cihazin ekran cozunurlugune gore ayarlanmasidir. Karsidaki cihazin ekran cozunurlugunu ise media query arayiciligiyla ogreniyoruz.
  • @media screen and (max-width 600px) {
    h1{ font-size: 12px;}
    }
    Maksimum ekran genisligi 600 ve daha asagisi olan bir cihaz siteye baglanirsa bu kod devreye girer ve h1 etiketi 12px olur.
  • max-width gibi parametreler icin verilen her bir css koduna breakpoint deniliyor.

SASS & SCSS

  • Bunlar CSS yazmayi kolaylastiran ara diller denilebilir.
  • SASS olarak yazilan kodlar SASS Compiler(Transpiler da denilebilir) ile CSS koduna cevrilip daha sonrasinda browser tarafindan taninirlar.
  • SASS ve SCSS arasinda ufak farkliliklar vardir fakat java mantigina daha benzer olan SCSS kullanimi daha kolaydir.

  • Variables

    • SCSS kullanirken sabit degiskenler olusturabiliriz, mesela renk kodlari gibi.
      ex: $color-primary: #ffffff;
      kullanim: background-color: $color-primary;

  • Nesting

    • Normal CSS kodunda etkilenecek her element icin ayri bir CSS kodu yazilir. Fakat SCSS kullanirken super class niteligindeki parent elementinin etiketi acilmissa child etiketleri icerisinde niteleyebilirsin.
    • body{
      font-size: 2px;
      p{
      color:$color-primary;
      }
      }
      Burada body icerisindeki p etiketleri degistirilir. body nitelenip daha sonrasinda body p olarak nitelemeye gerek kalmaz.

    • Nesting ise eger alt elemenlere degil de kendi ozelliklerine erismek istenirse this ozelligini kullanmak gibi &(ampersand) isareti kullanilir.
    • body{
      font-size: 2px;
      &:hover{
      color:$color-primary;
      }
      }
      Burada bodye hover olayi gerceklestiginde renk, color-primary icerisine atanmis renge cevrilir. Kisacasi & isareti, this gorevi gorur.

  • Mixin & Extend

    • Mixin ve Extend temelinde benzer ozellikler fakat buyuk farkliliklar bulundurmaktadirlar.
    • Kisa ornek vermek gerekirse Mixin fonksiyon gibi parametre alabilir ve bu parametrelere gore ozellikler degisebilir. Extend ise verilen etikete direkt icerisindeki tum ozellikleri verir fakat bu ozellikler degistirilemezler.
    • Degistirilebilecek ortak olan ozellikleri Mixin ile belirlemek ve degistirilmeyecek ortak kodlari ise Extend ile etikete vererek kod tekrarindan kacinmis oluruz.
    • box ornegini verecek olursak;
      @mixin box($color){
      width:100px;
      height: 100px;
      color: $color;
      }
      #first-box{
      @include box(#333);
      }
      Sekildeki gibi bir ornekte, mixin ile icerisindeki sabit ve degisken degerler olusturuluyor. Degisken olarak color, sabit olarak da width ve height degerleri veriliyor. Asagida @include etiketiyle box mixini etikete dahil edilip parametre olarak degisken olan ozellikler yollaniyor.
    • box ornegini verecek olursak;
      .box {
      width:100px;
      height: 100px;
      color: #333;
      }
      #first-box{
      @extend .box;
      }
      Bu ornekte box icerisindeki tum ozellikler first-box idsine aktarilir ve degistirilemez. Ortak ve sabit degerlerde kod tekrarindan kurtulmak icin guzel bir yontem.

Flexbox

  • Flexbox; display,float gibi yerlestirme ozelliklerinde yasanan sikintilardan arindirilmis esnek bir display ozelligidir.
  • Flexbox ozelligi iki kisimdan olusur. Flex container ve flex item. Flex container, itemleri icinde barindiran yapidir. Flex item ise container icerisindeki her bir elemandir.
  • Flex container ve flex item'in farkli degiskenleri vardir. Sol tarafta flex container ozellikleri, sag tarafta ise flex item ozellikleri bulunmaktadir.

  • Flex Container

    • display ozelligine flex degeri atanmasi gerekir.
    • Icerisindeki bulunan elemanler block ozelligini kaybedip inline-block gibi davranirlar.
    • Icerisindeki elemanlari istedigimiz gibi hareket ettirebiliriz. Bunun icin bircok degisken var.
    • flex-direction : itemlerin nasil dizilecegini belirler. row(stardart),row-reverse,column,column-reverse
      flex-wrap : sigmayan itemlerin asagiya kaydirilmasi. nowrap(stardart),wrap,wrap-reverse

  • Flexbox Ailgnments

    • Hizalama islemleri
    • justify-content: elemanlari yatayda hizalamaya yarar.
      • center
      • flex-start : sola dogru
      • flex-end : saga dogru
      • space-between : her elemanin arasinda esit bosluk olur,en bastaki ve en sondaki bosluklar ayarlanmaz. sadece elemanlarin arasindaki bosluklar ayarlanir
      • space-around : space-between ile benzerdir fakat en bastaki ve en sondaki bosluklari da ayarlar. en basta ve en sonda x bosluk varsa elemanlar arasi 2x bosluk olur
      • space-evenly : en bastaki ve en sondaki bosluklar dahil butun bosluklar esit olarak ayarlanir.
    • align-items: elemanlari dikeyde hizalamaya yarar.
      • stretch (standart): yukseklikler belirtilmemisse butun itemlar ayni yukseklikte olur. birine verilen yukseklik hepsini etkiler.
      • center : merkeze sabitler
      • flex-start : en uste hizalanir
      • flex-end : en alta hizalanir
      • baseline : iceriklerin alt sinirindan hizalama yapilir. ornekte icerik olarak yazilan yazilarin tam altintan hizalama yapilmis.
    • flex-direction : row icin justify-content yatayda hizalama yapar. align-items dikeyde hizalama yapar.
    • flex-direction : column icin justify-content dikeyde hizalama yapar. align-items yatayda hizalama yapar.

  • Flex Items

    • Flex container icerisindeki elemanlarin, eleman seviyesinde siralama ve hizalama islemleri burada yapilir.
    • Flex ozelligi olan container icerisindeki itemlarin classlarina verilebilecek ozellikler:
      • align-self : itemin kendi konumu container icerisinde ayarlanabilir.
      • order : elemanin container icerisindeki sirasini belirtir. o siraya gore itemlar dizilir. negatiften pozitife dogru bir oncelik vardir. containeri yukarida verilmis xy grafigi gibi dusun. negatif olan en solda olur.
      • flex-grow : elemanin sayfada kapladigi alan ile alakalidir. tum elemanlara 1 atanmis olsaydi 2 atanan eleman digerlerinden 2 kat daha fazla yer kaplayacakti. Eger sadece bir tanesine herhangi bir deger atanmis olsaydi deger atanan eleman diger elemanlardan kalan tum alani kaplardi.
        Ornekte item3 ve item4 flex-grow 1, item2 4, item1 ise 2 degerine sahiptir.
      • flex-basis : yuzde ya da px deger verilebilir. itema verilen deger kadar containerda yer kaplar.
      • flex-shrink : grow buyumekle alakali fakat shrink daralmakla alakali. varsayilan degeri 1'dir. kendine ayrilan yer kalmadiysa otomatik olarak kuculmeye gider. Eger bu degeri 0 yaparsak alan kalmasa bile kuculmez scroll bar cikar.
      • Son uc ozelligi tek tek atamak yerine tek bir seferde atama yapilabiliyor.
        flex: (grow) (shrink) (basis) seklinde bir kullanim yapilabilir. Eger flex: x seklinde bir kullanim yapilirsa sadece grow icin kullanilir.

  • Row Alignment

    • Container icerisinde birden fazla satir varsa bunlarin hizalanmasini align-content ile yapariz.
    • Alabilecegi degerler justify-content ile aynidir.

  • Flex & Margin

    • Flex container icerisinde margin kullanimi gayet esnek bir yapidadir.
    • Eger margin : auto ozelligi verilirse verilen item containerin tam ortasina yerlesir
    • Eger margin-left : auto ozelligi verilirse verilen item containerin sagina yapisir
    • Eger margin-right : auto ozelligi verilirse verilen item containerin soluna yapisir
    • Eger margin-top : auto ozelligi verilirse verilen item containerin altina yapisir
    • Eger margin-bottom : auto ozelligi verilirse verilen item containerin ustune yapisir
